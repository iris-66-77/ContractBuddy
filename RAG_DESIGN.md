## 二、RAG知识库核心功能

### 2.1 知识库分类体系

```
知识库/
├── 合同案例库
│   ├── 租房合同
│   ├── 劳动合同
│   ├── 购房合同
│   ├── 服务协议
│   └── 借款合同
│
├── 霸王条款模式库
│   ├── 免责条款
│   ├── 违约金过高
│   ├── 最终解释权
│   ├── 单方解除权
│   └── 押金不退
│
├── 法律法规库
│   ├── 民法典合同编
│   ├── 劳动合同法
│   ├── 消费者权益保护法
│   └── 相关司法解释
│
└── 谈判话术库
    ├── 常见问题回应
    ├── 法律条文引用
    └── 成功案例
```

### 2.2 向量检索增强分析流程

1. **合同文本分析**
   - 识别关键条款
   - 向量化查询
   - 检索相似霸王条款
   - 查找相关法律条文

2. **相似案例推荐**
   - 基于合同类别检索
   - 基于风险等级检索
   - 基于条款内容检索

3. **动态分析增强**
   - RAG补充法律依据
   - 参考历史案例分析
   - 提供更专业建议

## 三、识别分析能力增强方案

### 3.1 增强分析能力

| 分析维度 | 原有能力 | 增强能力 |
|---------|---------|---------|
| 条款识别 | 简单正则 | 模式匹配 + 向量检索 |
| 风险评级 | LLM直接输出 | 综合模式库评分 + LLM验证 |
| 法律依据 | LLM生成 | RAG检索真实法条 + LLM整合 |
| 谈判建议 | 通用建议 | 历史案例检索 + 个性化建议 |
| 相似推荐 | 无 | 语义相似合同推荐 |

## 五、数据结构设计

### 5.1 向量文档结构

```javascript
{
  id: string,
  content: string,
  metadata: {
    type: 'clause'|'law'|'case',
    category: string,
    riskLevel: 'low'|'medium'|'high'|'critical',
    source: string,
    tags: string[],
    createdAt: date
  },
  embedding: number[]
}
```

### 5.2 检索结果结构

```javascript
{
  query: string,
  results: [
    {
      id: string,
      content: string,
      metadata: object,
      score: number,
      highlight: string
    }
  ],
  total: number
}
```
